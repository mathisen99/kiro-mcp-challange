# Final README guide

The starter README intentionally says the implementation is incomplete. Replace
that status before submission.

The final README should contain:

1. project pitch;
2. demo GIF or video link;
3. what MCP connects to;
4. why Kiro could not perform the battle without this integration;
5. architecture diagram;
6. Java and Arch Linux prerequisites;
7. clone and build commands;
8. first direct smoke test;
9. Kiro MCP configuration;
10. every MCP tool with examples;
11. viewer connection steps;
12. replay fallback;
13. security warning about executable bots;
14. test commands;
15. troubleshooting;
16. licenses and attribution;
17. known limitations.

Never include an unverified command.
