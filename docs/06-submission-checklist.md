# Submission checklist

## Functionality

- [ ] custom MCP server is authored in this repository
- [ ] real Battle Runner dependency is used
- [ ] two real bots launch
- [ ] real score components return
- [ ] failure is never replaced with fake data
- [ ] Kiro can invoke the tools
- [ ] live viewer or successful official-GUI replay playback proves visual execution

## Repository

- [ ] public GitHub repository
- [ ] `.kiro/settings/mcp.json` committed
- [ ] README setup verified
- [ ] Gradle Wrapper committed
- [ ] source and tests committed
- [ ] chosen license committed
- [ ] dependency versions pinned
- [ ] no absolute home paths
- [ ] no secrets
- [ ] no recordings, logs, or build output

## Documentation

- [ ] what the server connects to
- [ ] why MCP is necessary
- [ ] architecture
- [ ] prerequisites
- [ ] installation
- [ ] tool examples
- [ ] viewer instructions
- [ ] third-party viewer attribution and replay instructions
- [ ] testing commands
- [ ] untrusted bot warning
- [ ] attribution

## Video

- [ ] end-to-end flow
- [ ] Kiro MCP connection visible
- [ ] real battle visible
- [ ] real results visible
- [ ] approximately 30–60 seconds
- [ ] maximum 3 minutes

## Final

- [ ] links entered in `SUBMISSION.md`
- [ ] clean checkout verification completed
- [ ] official challenge page rechecked
- [ ] submission completed before deadline
